# 网易 UU 远程「仅观看」实现方案研究报告

> 调研日期：2026-05-16
> 目标：在本地（主控端）使用 UU 远程时，让画面正常显示，但鼠标/键盘信号不发送到被控端，达到"只看不操作"的效果。

---

## 一、关键结论（先看这里）

**UU 远程官方已经内置了「观看模式」功能**，无需自己开发或反编译。

- 官方功能名：**观看模式**（与「操作模式」相对）
- 功能描述：仅查看被控端屏幕，不进行操作，控制端**无法操控键鼠**，避免误操作。
- 适用场景：远程监控设备状态（如查看下载进度、看家里电脑是否正常运行等）。

但是：

- 官方文档对该功能的**明确开启步骤主要见于 macOS 内测版本 (4.6.101)**。
- **Windows 端**是否在最新版本中已经放出该开关、放在哪里，官网帮助页面**没有专门的图文教程**，需要在客户端 UI 中实际确认。

---

## 二、推荐的尝试顺序

### 方案 A：使用官方观看模式（首选，零成本）

操作步骤（请你在主控端 Windows 电脑上动手验证）：

1. **升级到最新版 UU 远程**
   - 官网下载页：https://uuyc.163.com/
   - 当前流通版本号约 4.16.x（PC 版），macOS 内测已到 4.6.101。
   - 如果 Windows 稳定版没有，可以试一下官网是否有 Windows 内测版 / Beta 通道。

2. **发起远程控制会话**，连上被控端。

3. **在控制窗口顶部工具栏上**寻找以下任一入口：
   - 「**观看模式 / 操作模式**」切换按钮（可能是一个眼睛图标或下拉菜单）
   - 工具栏「更多」/「设置」/「⋯」菜单里的子项
   - 与"游戏鼠标"（Ctrl+Alt+Enter）按钮相邻的位置

4. 如果找到：直接切换即可，**任务完成，不需要写任何代码**。

5. 如果在 UI 上完全找不到：
   - 在客户端右下角「反馈」入口直接问官方："Windows 端是否支持观看模式？如何开启？"
   - 官方反馈通常 1-2 个工作日回。

### 方案 B：自己写本地拦截程序（仅当方案 A 不可行）

只有当确认 Windows 客户端确实没有该功能、且官方短期不会提供时，再走这条路。

详细需求见本文档第四节，**不需要反编译 UU 远程**。

## 四、备选方案 B：本地拦截程序详细需求

> 仅在官方观看模式确实不可用时启动此方案。

### 4.1 技术原理（黑盒，不需要反编译）

利用 Windows 系统级低级钩子 (`SetWindowsHookEx` + `WH_MOUSE_LL` / `WH_KEYBOARD_LL`)，在操作系统把键鼠输入消息派发给 UU 远程窗口**之前**就拦截掉。UU 远程的进程根本接收不到这些消息，自然就不会发到远端。

- **优点**：不碰 UU 远程进程内存，不触发反作弊；UU 升级也不会失效。
- **缺点**：需要你以管理员权限运行这个小工具。

### 4.2 所需工具

| 工具 | 用途 | 是否必需 |
|------|------|---------|
| Spy++ / WinSpy / ViewWizard | 探测 UU 远程渲染窗口的类名(Class Name) | **必需** |
| Python 3.10+ 或 Visual Studio (C++/C#) | 编译运行拦截程序 | **必需**（推荐 Python，最快） |
| `pynput` 或 `pyWinhook` 库（Python） | 简化键鼠钩子调用 | Python 路线必需 |
| 管理员权限 | 安装全局钩子需要 | **必需** |

### 4.3 你需要给我的数据（用 Spy++ 抓取）

1. UU 远程**实际渲染远端画面的那个子窗口**的：
   - Class Name（例：`UURemoteRenderWnd` 之类）
   - Window Text（标题，可能为空）
   - 父窗口类名（用于多层确认）
2. UU 远程主进程名（任务管理器里的 `XXX.exe`，例：`uu_remote.exe`）

### 4.4 程序功能清单（我可以代写）

- [x] 注册全局热键（默认 `Ctrl+Alt+V`）：一键切换"观看模式 / 正常模式"
- [x] 低级鼠标钩子：识别当前光标下窗口，若是 UU 远程渲染窗口则丢弃移动/点击/滚轮消息
- [x] 低级键盘钩子：当 UU 远程窗口处于前台时丢弃所有按键消息（热键本身豁免）
- [x] 光标平滑优化：丢弃 `WM_MOUSEMOVE` 的同时调用 `SetCursorPos`，避免本地光标"撞墙"
- [x] 系统托盘图标 + 状态提示（当前是观看还是控制）
- [x] 退出时干净卸载钩子

### 4.5 我能为你做什么

| 任务 | 我能做 | 需要你做 |
|------|--------|---------|
| 写 Python 拦截脚本 | ✅ 可以直接写出完整代码 | 你跑 Spy++ 给我窗口类名 |
| 写 C# / C++ 版本（更低延迟） | ✅ 可以 | 同上 |
| 自动探测 UU 窗口类名 | ⚠️ 可以写一个探测脚本，让你点一下 UU 画面就自动抓取 | 你运行一次 |
| 自动反编译 UU 找内部 API | ❌ 不推荐，违反 ToS 且风险高 | — |
| 直接在你电脑上执行安装 | ❌ 我不能远程操控你的电脑 | 你本地双击运行即可 |

## 五、风险与合规提示

1. **反编译/HOOK UU 进程内存** → 高风险（可能违反用户协议，可能触发反作弊）→ **本方案不采用**。
2. **系统级键鼠钩子（方案 B）** → 低风险，是 Windows 公开 API，杀软可能误报需要加白名单。
3. 任何方案都建议先**用一台不重要的被控机器**测试，避免误操作生产环境。

---

## 六、下一步建议（按优先级）

1. ⭐ **你先升级 Windows 端 UU 到最新版**，连上被控端，截图工具栏发我 → 我帮你判定方案 A 是否可行。
2. 如果方案 A 不行 → 跑 Spy++ 抓窗口类名给我 → 我给你写方案 B 的 Python 原型代码。
3. 原型跑通后再决定是否需要打包成 .exe / 加托盘 / 加快捷键管理。
