# 🎮 UU 仅观看模式 (UU View-Only Toggle)

> ⭐ **如果觉得项目有用，请点击右上角 Star 支持一下！**

为网易 UU 远程添加真正的**仅观看模式**，防止本地鼠标/键盘操作转发到被控端，同时屏幕正常显示。

## 🖼️ 界面

```
┌──────────────────────────┐
│                          │
│          ●  (绿色=控制中) │
│   已找到 UU 远程窗口      │
│   快捷键: Ctrl+Alt+V     │
│                          │
│   ┌──────────────────┐   │
│   │   启用仅观看     │   │
│   └──────────────────┘   │
│                          │
│      ┌──────────┐        │
│      │ 重新扫描  │        │
│      └──────────┘        │
│   关闭自动恢复控制       │
└──────────────────────────┘
```

## ⚙️ 原理

采用**双重拦截机制**，确保本地输入不会转发到被控端：

### 1️⃣ 禁用子窗口（消息层面）

通过 Win32 `EnableWindow` API 禁用 UU 远程窗口（`GameViewer.exe`）的所有子窗口，使 Windows 在消息层面丢弃鼠标/键盘输入。顶层窗口保持启用，可以拖拽/关闭。

### 2️⃣ 透明遮罩窗口（物理层面）

在每个 UU 窗口上方创建一个透明的遮罩窗口（`WS_EX_LAYERED`，alpha=1），在物理层面拦截所有鼠标点击，防止点击穿透到 UU 窗口。

遮罩不设置 `WS_EX_TOPMOST`，而是通过 `GetWindow(GW_HWNDNEXT)` 动态查找 UU 窗口上方的可见窗口，将遮罩插入其下方，从而**仅遮挡 UU 窗口，不干扰其他重叠窗口**。

遮罩跟随 UU 窗口的位置/大小变化（1 秒轮询），UU 最小化时自动隐藏遮罩。

> ✨ 无需管理员权限，无需 DLL 注入，无需全局钩子。

## 🚀 使用方式

1. 📥 下载 `dist/UUViewOnly.exe`
2. ▶️ 双击运行，无需安装 Python 或任何依赖
3. 🔍 程序启动后自动扫描 UU 远程窗口

### 🔧 重新生成 EXE

如需从源码重新构建 exe：

1. 安装 Python 3.10+
2. 双击 `build_exe.bat`（自动安装依赖并打包）
3. 在 `dist\` 目录找到新生成的 `UUViewOnly.exe`

### 📋 操作

| 操作 | 说明 |
|------|------|
| 🟢 点击「启用仅观看」 | 切换到仅观看模式，本地输入不转发 |
| 🔴 点击「恢复控制」 | 恢复正常控制模式，移除遮罩 |
| ⌨️ Ctrl+Alt+V | 快捷键切换模式 |
| 🔄 Ctrl+Alt+B | 重新扫描窗口（UU 窗口重建后使用） |
| ❌ 关闭按钮 | 退出程序，自动恢复控制模式并移除遮罩 |

## ⚠️ 注意事项

- 仅观看模式下，UU 自身工具栏按钮（截图、分辨率等）也会被禁用，需先恢复控制
- 如果 UU 窗口关闭后重新打开，点击「重新扫描」按钮即可
- 程序退出时自动恢复控制模式并销毁所有遮罩窗口
- 遮罩仅覆盖 UU 窗口区域，不影响其他窗口的正常交互
